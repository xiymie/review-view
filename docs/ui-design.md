# 页面设计文档

## 整体布局

左右分栏：左侧固定侧边栏（240px）+ 右侧内容区。

- 侧边栏深色背景（`#1F2937`），内容区浅灰背景（`#F7F8FA`）
- 侧边栏包含应用名称 + 6 个一级导航入口，当前页高亮（绿色 `#22C55E`）
- 二级页面通过面包屑导航返回上级，格式：`上级页面名 / 当前页面名`

技术栈：Vue 3 + Element Plus，构建产物通过 `go:embed` 打包进二进制。

## 导航结构

侧边栏一级入口：

| 名称 | icon | 路由 |
|------|------|------|
| 仪表盘 | layout-dashboard | `/` |
| 项目 | folder | `/projects` |
| 模型配置 | cpu | `/models` |
| 仓库凭据 | key | `/credentials` |
| 任务 | list-checks | `/tasks` |
| 设置 | settings | `/settings` |

## 字体

- 全局字体：Inter，正文 13px，表头/辅助 12px，表单 14px
- 标题：28px bold（页面标题），18px semibold（区域标题）
- 数值强调：32px bold（统计卡片数值）

## 组件规范

### 按钮

- 主按钮（Primary）：绿色背景 `#22C55E`，白色文字，圆角 8px，padding 10px 18px
- 次要按钮（Secondary）：白色背景，灰色边框 `#D1D5DB`，深灰文字

### 卡片

- 白色背景，圆角 12px，外阴影 `0 1px 3px rgba(0,0,0,0.05)`
- 内边距 20-32px

### 表格

- 白色背景卡片容器，圆角 12px
- 表头：浅灰背景 `#F9FAFB`，12px 粗体灰色文字，padding 14px 16px
- 数据行：白色背景，padding 16px，行间 1px 分割线 `#F3F4F6`
- 状态列使用圆角药丸标签（border-radius: 999px）

### 表单

- 表单卡片宽度 `min(640px, 100%)`，grid 布局
- 标签 14px，输入框白色背景、1px 灰色边框、圆角 8px、padding 11px 12px
- 单选按钮组（inline-options）：flex 布局，radio 重置为 auto 宽度
- Checkbox（toggle-row）：flex 布局居中对齐，checkbox 重置为 auto 宽度
- 按钮组右对齐，flex + gap 12px

### 信息条（info-strip）

- flex 横排展示，wrap，gap 18px，padding 18px 20px，灰色文字

## 各页面

### 1. 仪表盘 `/`

- 页面标题"仪表盘" + 日期芯片
- 4 个统计卡片横排等宽：项目总数 / 运行中 / 今日完成 / 失败
- "最近任务"区域 + "查看全部"链接
- 最近任务表格：项目名 / Commit 范围 / 状态 / 触发方式 / 创建时间

### 2. 项目列表 `/projects`

- 页面标题 + 右侧"新建项目"主按钮
- 表格：名称（可点击进入详情）/ 仓库地址 / 分支 / 最近 Review / 最后任务状态药丸 / 操作（详情 + 删除）

### 3. 项目详情 `/projects/:id`

- 面包屑：项目 / 项目名
- 页面标题 + 副标题（仓库地址 + 分支）
- 右侧按钮组："编辑"（次要）+ "手动触发审核"（主按钮）+ "删除项目"（红色危险按钮，需确认）
- 信息条：模型配置名 / 溢出策略 / 最近 Review Commit
- 自定义 Prompt 展示卡片（有值时显示）
- 审核任务表格：ID（可点击）/ 状态药丸 / Commit 范围 / 触发方式 / 创建时间 / 操作（查看）

### 4. 新建项目 `/projects/new`

- 面包屑：项目 / 新建项目
- 表单字段：项目名称 / 仓库地址 / 仓库凭据（下拉，默认"无（公开仓库）"） / 分支（默认 main）/ 模型配置（下拉）/ 自定义 Prompt（textarea）/ 溢出策略（radio：排队等待/拒绝）/ 任务超时（分钟）
- 按钮：取消 + 创建项目

### 5. 编辑项目 `/projects/:id/edit`

- 面包屑：项目 / 项目名 / 编辑
- 表单结构同新建，预填现有数据
- 按钮：取消（返回详情页）+ 保存修改

### 6. 模型配置列表 `/models`

- 页面标题 + 右侧"新建配置"主按钮
- 表格：名称（可点击进入编辑）/ 平台药丸 / 模型 / Thinking 状态 / 创建时间 / 操作（编辑按钮）

### 7. 新建模型配置 `/models/new`

- 面包屑：模型配置 / 新建配置
- 表单根据平台类型动态切换字段组：
  - **API 类型**（openai/anthropic/ollama/deepseek/gemini/mistral）：配置名称 / 平台类型（下拉）/ API 地址 / API Key / 模型名称 / 最大上下文 Token（默认 32000）/ 启用 Thinking（checkbox）
  - **claude_cli 类型**：配置名称 / CLI 路径（默认 claude）/ 环境变量 JSON（textarea，默认 `{}`)/ Max Turns
- 公共字段：Review Prompt（textarea）
- 按钮：测试连接 + 取消 + 创建配置
- 测试结果区域（动态显示）

### 8. 编辑模型配置 `/models/:id/edit`

- 面包屑：模型配置 / 配置名
- 表单结构同新建，预填现有数据
- 按钮：测试连接 + 取消 + 保存修改

### 9. 任务列表 `/tasks`

- 页面标题
- 任务表格：项目名 / Commit 范围 / 状态（彩色药丸）/ 触发方式 / 创建时间 / 操作（查看 + 条件按钮：running 显示取消，failed/cancelled 显示重试）

### 10. 任务详情 `/tasks/:id`

- 面包屑：任务 / #ID 详情
- 页面标题 + 状态药丸（彩色）
- 信息条：Commit 范围 / Token 消耗（输入 N / 输出 N）/ 创建时间 / 开始时间 / 结束时间
- 独立日志区域（card）：按时间展示日志列表，每条包含时间戳 / 级别标签（info 蓝色 / warn 黄色 / error 红色）/ 消息，运行中任务通过 SSE 实时追加并自动滚动
- Tab 切换区域（2 个 tab）：
  - **Review 结果**（默认）：Markdown 渲染，运行中任务通过 SSE 实时显示流式结果
  - **Diff 内容**：pre 格式化展示

### 11. 全局设置 `/settings`

- 表单字段：最大并发任务数 / 全局溢出策略 / 任务超时时间 / 仓库根目录
- 按钮：保存设置

### 12. 凭据列表 `/credentials`

- 页面标题"仓库凭据" + 右侧"新建凭据"主按钮
- 表格：名称（可点击进入编辑）/ 用户名 / 创建时间 / 操作（编辑按钮）

### 13. 新建凭据 `/credentials/new`

- 面包屑：仓库凭据 / 新建凭据
- 表单字段：名称 / 用户名 / 密码或 Token（password 输入框）
- 按钮：取消 + 创建

### 14. 编辑凭据 `/credentials/:id/edit`

- 面包屑：仓库凭据 / 凭据名
- 表单字段：名称 / 用户名 / 密码或 Token（留空不修改）
- 按钮：取消 + 保存修改
- 删除凭据按钮（红色危险按钮，需确认），被项目引用时返回错误

## 前端交互

### 平台类型切换

模型配置表单中，选择不同平台类型时通过 JS 动态显示/隐藏字段组：
- `data-provider-group="api"`：API 类型字段
- `data-provider-group="claude_cli"`：CLI 类型字段

选择 `claude_cli` 时隐藏 API 字段组，显示 CLI 字段组；反之亦然。

### 测试模型连接

点击"测试连接"按钮：
1. 通过 JS fetch POST `/api/models/test`，提交当前表单数据
2. 按钮显示"测试中..."，结果区域动态展示
3. 成功：绿色文字显示连接成功 + LLM 响应
4. 失败：红色文字显示错误信息

### 任务实时更新

任务详情页中，当任务状态为 running 或 pending 时：
1. 通过 `data-task-id` 和 `data-task-status` 属性识别任务
2. 建立 SSE 连接 `GET /api/tasks/:id/stream`
3. 接收 `log` 事件追加到日志列表，自动滚动到底部
4. 接收 `result` 事件，用 `marked.parse()` 渲染累积 Markdown 到 `#review-result`
5. 接收 `done` 事件后关闭连接，渲染最终结果并更新状态标签

已完成任务的页面加载时，`#review-result` 中的原始 Markdown 由 `marked.parse()` 在客户端渲染。
